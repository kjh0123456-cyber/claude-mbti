## 1. 프로젝트 개요

- **게임 제목 (가칭)**: PyTetris
- **장르**: 클래식 퍼즐 (테트리스)
- **플랫폼**: PC (Windows / macOS / Linux 데스크탑)
- **타겟 유저**: Python 학습자, 게임 개발 입문자, 클래식 테트리스를 즐기는 캐주얼 게이머 (연령 무관)
- **핵심 콘셉트 한 줄 요약**: Python 기반으로 구현한 클래식 테트리스 게임으로, 점수 랭킹 시스템을 통해 도전 의욕을 자극하는 싱글플레이 퍼즐 게임
- **게임 비전**: 테트리스 원작의 핵심 재미 요소를 충실히 재현하되, Python 생태계 내에서 완결성 있는 프로젝트로 완성한다. 학습 목적과 플레이 재미를 동시에 충족하는 포트폴리오 수준의 결과물을 목표로 한다.
- **수익화 모델**: 없음 (개인/학습용 오픈소스 프로젝트)
- **예상 개발 기간**: 4~6주 (1인 개발 기준)
- **참고/경쟁 타이틀**:
  - **Tetris (원작)**: 7가지 테트로미노, 줄 제거, 레벨업 구조를 그대로 계승. 차별점은 Python 구현을 통한 코드 가독성과 확장 가능성
  - **Tetris Effect**: 화려한 비주얼은 제외하고, 원작 게임플레이 구조만 차용
  - **pygame-tetris (오픈소스)**: 유사 프로젝트 대비 로컬 랭킹 저장 기능을 추가하여 반복 플레이 동기 부여

## 2. 핵심 기능 목록

---

### 기능 1: 테트로미노 생성 및 낙하

- **설명**: 게임 시작 시 무작위로 테트로미노(I, O, T, S, Z, J, L 7종)를 생성하고, 일정 속도로 아래 방향으로 자동 낙하시킨다.
- **세부 사항**:
  - 낙하 속도는 레벨에 따라 증가한다. 레벨 1 기준 0.8초에 한 칸, 레벨 10 이상에서는 0.05초에 한 칸으로 선형 감소한다.
  - 블록 색상은 테트로미노 종류별로 고정한다 (I: 하늘색, O: 노란색, T: 보라색, S: 녹색, Z: 빨간색, J: 파란색, L: 주황색).
  - 낙하 중인 블록이 바닥 또는 다른 블록에 닿은 후 0.5초 내에 추가 입력이 없으면 블록이 고정(Lock Down)된다.
  - 다음에 등장할 테트로미노를 별도 프리뷰 영역(Next Block)에 1개 미리 표시한다.
- **유저 가치**: 예측 가능한 블록 순서와 시각적 힌트를 제공하여 전략적 배치를 유도하고 게임 몰입도를 높인다.
- **의존 기능**: 기능 3 (충돌 감지), 기능 4 (줄 제거), 기능 5 (점수/레벨 시스템)
- **우선순위**: 🔴 필수

---

### 기능 2: 블록 조작 (이동 및 회전)

- **설명**: 키보드 입력으로 낙하 중인 테트로미노를 좌우 이동, 시계 방향 회전, 소프트 드롭(가속 낙하), 하드 드롭(즉시 착지)할 수 있다.
- **세부 사항**:
  - 좌우 이동: 방향키 Left/Right. 벽에 닿으면 이동 불가.
  - 회전: 방향키 Up 또는 Z 키 (시계 방향 90도). 벽이나 다른 블록과 겹칠 경우 Wall Kick 로직을 적용하여 1칸 이내 보정을 시도한다.
  - 소프트 드롭: 방향키 Down. 낙하 속도를 현재의 10배로 증가시키며, 소프트 드롭으로 이동한 칸 수만큼 1점씩 추가 점수를 부여한다.
  - 하드 드롭: Space 키. 블록을 즉시 최하단 착지 위치로 이동시키고 고정한다. 이동 칸 수 × 2점 추가 점수를 부여한다.
  - Ghost Piece: 블록이 착지할 위치를 반투명하게 미리 표시한다 (색상은 동일하되 알파값 40% 적용).
  - 키 반복 딜레이(DAS, Delayed Auto Shift): 첫 입력 후 170ms 딜레이, 이후 50ms 간격으로 반복 이동.
- **유저 가치**: 정밀한 블록 배치가 가능하고, 고급 조작 기술(T-스핀, 하드드롭)을 구사할 수 있어 숙련도에 따른 성취감을 제공한다.
- **의존 기능**: 기능 1 (테트로미노 생성), 기능 3 (충돌 감지)
- **우선순위**: 🔴 필수

---

### 기능 3: 충돌 감지 및 보드 관리

- **설명**: 테트로미노가 이동하거나 회전할 때 게임 보드 경계 및 고정된 블록과의 충돌을 감지하고, 불가능한 동작을 차단한다.
- **세부 사항**:
  - 게임 보드 크기: 가로 10칸 × 세로 20칸 (표준 테트리스 규격).
  - 충돌 감지는 테트로미노를 구성하는 각 셀 좌표 기준으로 매 프레임마다 수행한다.
  - 블록 고정 시, 해당 셀 좌표를 보드 2차원 배열에 기록하고 색상 정보도 함께 저장한다.
  - 새 블록 생성 시 스폰 위치(보드 상단 중앙, X=4, Y=0)에 이미 고정된 블록이 존재하면 게임 오버를 트리거한다.
- **유저 가치**: 정확한 물리 규칙이 보장되어야 게임의 공정성과 신뢰감이 형성된다.
- **의존 기능**: 기능 1, 기능 2, 기능 4
- **우선순위**: 🔴 필수

---

### 기능 4: 줄 제거 및 시각 효과

- **설명**: 가로 한 줄이 빈 칸 없이 채워지면 해당 줄을 제거하고, 위의 블록들을 아래로 내린다. 제거된 줄 수에 따라 점수를 산정한다.
- **세부 사항**:
  - 줄 제거 판정: 매 블록 고정 직후, 보드 전체를 검사하여 가득 찬 줄 목록을 추출한다.
  - 동시 제거 가능 줄 수: 최대 4줄 (테트리스).
  - 줄 제거 연출: 제거 대상 줄을 흰색으로 0.1초간 깜빡인 뒤 삭제하고, 위 블록을 내린다.
  - 점수 산정 (레벨 N 기준):
    - 1줄: 100 × N
    - 2줄: 300 × N
    - 3줄: 500 × N
    - 4줄 (테트리스): 800 × N
  - 콤보: 연속으로 줄을 제거할 경우 콤보 카운터가 증가하며, 콤보 횟수 × 50 × N 점수를 추가 부여한다.
- **유저 가치**: 한 번에 여러 줄을 제거하는 쾌감과 콤보 보너스를 통해 고득점 전략을 추구하는 재미를 제공한다.
- **의존 기능**: 기능 3 (충돌/고정), 기능 5 (점수 시스템)
- **우선순위**: 🔴 필수

---

### 기능 5: 점수 및 레벨 시스템

- **설명**: 게임 진행에 따라 점수가 누적되고, 일정 줄 수 제거 시 레벨이 오르며 블록 낙하 속도가 빨라진다.
- **세부 사항**:
  - 레벨업 조건: 누적 제거 줄 수가 레벨 × 10에 도달할 때마다 1레벨 증가 (예: 레벨 1 → 2는 10줄, 레벨 2 → 3은 20줄).
  - 최대 레벨: 15 (이후 속도 고정, 점수는 계속 누적).
  - 게임 화면에 현재 점수, 레벨, 제거 줄 수를 실시간으로 표시한다.
  - 레벨업 시 0.5초간 화면에 "LEVEL UP!" 텍스트를 표시한다.
- **유저 가치**: 점진적인 난이도 상승이 도전감을 유지시키고, 점수 경쟁을 통해 재플레이 의욕을 높인다.
- **의존 기능**: 기능 4 (줄 제거), 기능 6 (랭킹 시스템)
- **우선순위**: 🔴 필수

---

### 기능 6: 로컬 랭킹 시스템

- **설명**: 게임 오버 시 획득한 점수를 로컬에 저장하고, 상위 10개의 기록을 랭킹 화면에서 조회할 수 있다.
- **세부 사항**:
  - 저장 형식: JSON 파일 (`scores.json`)을 게임 실행 디렉토리에 저장. 항목 구성: 닉네임(최대 10자), 점수, 도달 레벨, 기록 날짜/시간.
  - 게임 오버 시 플로우:
    1. "Game Over" 화면 표시 (획득 점수, 도달 레벨, 제거 줄 수 요약)
    2. 현재 점수가 기존 Top 10에 진입하면 닉네임 입력 창을 표시
    3. 닉네임 입력 후 저장 및 랭킹 화면으로 전환
  - 랭킹 화면: 순위, 닉네임, 점수, 레벨, 날짜를 테이블 형태로 표시. 현재 세션의 신규 기록은 색상으로 강조 표시.
  - 랭킹 데이터는 점수 내림차순 정렬. 동점 시 달성 시간이 빠른 기록을 우선.
  - 메인 메뉴에서 랭킹 화면을 별도로 조회할 수 있다.
- **유저 가치**: 자신의 과거 기록과 경쟁하는 구조가 반복 플레이를 유도하고, 점수 향상에 따른 성취감을 가시적으로 제공한다.
- **의존 기능**: 기능 5 (점수 시스템)
- **우선순위**: 🔴 필수

---

### 기능 7: 게임 상태 관리 (일시정지 / 재시작 / 게임 오버)

- **설명**: 게임 진행 중 일시정지, 재시작, 게임 오버 처리를 명확한 상태 전환 로직으로 관리한다.
- **세부 사항**:
  - 일시정지: P 키 또는 ESC 키로 토글. 일시정지 중 블록 이동 입력은 무시하며, 화면에 "PAUSED" 오버레이를 표시한다.
  - 재시작: 게임 오버 화면 또는 일시정지 화면에서 R 키로 즉시 재시작. 보드, 점수, 레벨을 초기화한다.
  - 메인 메뉴: 타이틀 화면 → 게임 시작 / 랭킹 보기 / 종료 메뉴 구성. 방향키와 Enter 키로 선택.
  - 게임 상태 흐름: `메인메뉴 → 게임 중 → 일시정지(선택) → 게임 오버 → 랭킹 입력(조건부) → 랭킹 화면 → 메인메뉴`
- **유저 가치**: 명확한 상태 전환이 유저의 혼란을 방지하고 편의성을 높인다.
- **의존 기능**: 기능 5, 기능 6
- **우선순위**: 🔴 필수

---

### 기능 8: Hold 기능

- **설명**: 현재 낙하 중인 테트로미노를 보관(Hold)하고, 이후 원하는 시점에 꺼내 사용할 수 있다.
- **세부 사항**:
  - C 키 또는 Shift 키로 Hold 실행.
  - Hold 슬롯은 1개. 이미 Hold된 블록이 있을 경우, 현재 블록과 교환된다.
  - 교환 직후에는 한 번 더 Hold 불가 (같은 블록의 연속 Hold 방지). 블록이 착지·고정되어야 다시 Hold 가능.
  - Hold 슬롯은 게임 화면 좌측에 별도 영역으로 표시.
- **유저 가치**: 전략적 블록 운용이 가능해져 고급 플레이를 지원하고, 위기 상황 탈출 수단으로 활용된다.
- **의존 기능**: 기능 1, 기능 3
- **우선순위**: 🟡 중요

## 3. 사용자 시나리오

### 시나리오 1: 신규 유저 첫 경험 (온보딩)

> 감정 상태: 호기심, 기대감

1. 프로그램을 실행하면 타이틀 화면이 표시된다. "PyTetris" 로고와 함께 "게임 시작 / 랭킹 / 종료" 메뉴가 보인다.
2. 유저가 Enter 키를 눌러 게임을 시작한다.
3. 화면 중앙 상단에서 첫 테트로미노(무작위)가 나타나고 천천히 낙하를 시작한다.
4. 유저는 방향키로 블록을 이동하고, 위 방향키로 회전하면서 기본 조작을 익힌다.
5. Ghost Piece 덕분에 블록이 어디에 착지할지 직관적으로 파악할 수 있다.
6. 몇 개의 블록을 쌓다가 처음으로 한 줄을 완성하자 줄이 제거되고 점수가 올라간다.
7. 블록이 점점 쌓여 화면 상단에 닿으면 게임 오버 화면이 표시된다. 현재 점수와 레벨이 요약되어 보인다.
8. 점수가 Top 10에 진입하면 닉네임 입력 창이 나타나고, 유저가 이름을 입력하면 랭킹에 기록된다.
9. 랭킹 화면에서 자신의 기록을 확인한 뒤, 메인 메뉴로 돌아온다.

---

### 시나리오 2: 핵심 게임플레이 루프 (숙련 유저)

> 감정 상태: 집중, 긴장감, 성취감

1. 유저가 게임을 시작하고, 레벨 1에서 블록을 빠르게 배치하기 시작한다.
2. Next Block 프리뷰를 보며 다음 블록을 미리 예측하고 현재 블록의 최적 위치를 결정한다.
3. I 블록이 나왔을 때 미리 쌓아둔 세로 공간에 하드 드롭(Space)하여 4줄을 동시에 제거한다. "TETRIS!" 연출과 함께 대량 점수가 획득된다.
4. 어울리지 않는 Z 블록이 연속으로 나오자 Hold(C 키)에 보관하고, 다음 블록을 먼저 사용한다.
5. 레벨 5에 도달하자 "LEVEL UP!" 텍스트가 표시되고 블록이 눈에 띄게 빨라진다.
6. 집중이 필요한 순간 P 키로 일시정지 후 잠시 휴식, 다시 P 키로 재개한다.
7. 레벨 10을 돌파하며 역대 최고 점수를 경신한다. 랭킹 1위로 이름이 올라가는 것을 확인하고 재도전 의지가 생긴다.

---

### 시나리오 3: 고득점 도전 및 랭킹 경쟁

> 감정 상태: 경쟁심, 반복 도전 욕구

1. 유저가 랭킹 화면을 확인하여 현재 1위 점수와 자신의 점수 차이를 파악한다.
2. 게임을 시작하고, 이번 회차는 1위 점수 초과를 목표로 의식적으로 플레이한다.
3. 소프트 드롭과 하드 드롭을 적극 활용하여 보너스 점수를 최대한 쌓는다.
4. 연속 줄 제거로 콤보 보너스를 발생시켜 점수를 빠르게 올린다.
5. 실수로 블록이 쌓여 위기 상황이 오자 Hold 기능으로 I 블록을 꺼내 위기를 탈출한다.
6. 게임 오버 후 랭킹을 확인하니 1위로 갱신되어 있다. 닉네임을 입력하고 기록을 저장한다.
7. 랭킹에서 자신의 이름이 1위에 오른 것을 보며 만족감을 느끼고 다음 도전을 계획한다.

## 4. 기술 스택

### 게임 엔진 / 핵심 라이브러리

- **추천**: `pygame 2.x` — 선택 이유: 게임 루프, 이벤트 처리, 2D 렌더링, 키보드 입력, 사운드를 통합 지원하는 Python 게임 개발 표준 라이브러리. 테트리스 수준의 2D 퍼즐 게임에 최적화되어 있으며, 프레임 단위 정밀 제어가 가능하다.
- **대안**: `tkinter` — Python 기본 내장 라이브러리로 별도 설치가 불필요하나, 게임 루프 및 애니메이션 처리가 pygame 대비 복잡하고 성능이 낮아 권장하지 않는다. 학습 목적의 초소형 프로젝트에만 적합하다.

### 프론트엔드 / 클라이언트

- **렌더링**: pygame Surface, Rect 기반 2D 타일 렌더링
- **폰트**: pygame.font 모듈 (시스템 폰트 또는 번들 TTF 폰트 파일)
- **해상도**: 기본 창 크기 480×640px (보드 영역 200×400px + UI 패널 영역). 고정 해상도 (리사이즈 미지원으로 구현 단순화)
- **색상 팔레트**: 테트로미노 7색 + 배경(#1a1a2e) + UI 텍스트(#ffffff, #f0c040)

### 백엔드 / 데이터 저장

- **데이터 저장**: Python 내장 `json` 모듈로 로컬 파일(`scores.json`) 읽기/쓰기. 서버 불필요.
- **데이터 구조 예시**:

```json
{
  "rankings": [
    {"name": "PLAYER1", "score": 45200, "level": 12, "lines": 98, "date": "2026-05-29T14:32:01"}
  ]
}
```

- **예외 처리**: 파일 없음/손상 시 빈 랭킹으로 초기화 후 재생성

### 프로젝트 구조 (권장)

```
pytris/
├── main.py              # 진입점, 게임 루프
├── game.py              # 게임 상태 관리 (GameState 클래스)
├── board.py             # 보드 데이터 및 충돌 감지
├── tetromino.py         # 테트로미노 정의, 회전, 이동
├── renderer.py          # pygame 렌더링 담당
├── score.py             # 점수 계산 로직

```

## 6. 리스크 및 고려사항

- **기술적 리스크**:
  - pygame의 프레임 타이밍 정밀도가 OS 환경(Windows/macOS/Linux)에 따라 미묘하게 다를 수 있다. `pygame.time.Clock.tick_busy_loop()`를 활용하거나 delta time 기반 낙하 속도 계산으로 완화한다.
  - JSON 파일 동시 접근 이슈는 싱글 프로세스 앱 특성상 발생하지 않으나, 비정상 종료 시 파일 손상 가능성이 있다. 저장 전 임시 파일에 먼저 쓰고 교체(atomic write)하는 방식으로 방지한다.

- **시장 리스크**:
  - 해당 없음 (학습용 개인 프로젝트). 다만 포트폴리오로 공개 시, 테트리스 상표권(Tetris Holding LLC)에 유의하여 게임 제목에 "Tetris"를 직접 사용하지 않는다.

- **운영 리스크**:
  - 1인 개발이므로 개발 지연 시 Phase 2, 3 기능의 범위를 조정한다. Hold 기능, 사운드는 MVP 완성 후 추가하는 것으로 우선순위를 낮게 유지한다.
  - Python 버전(3.9 / 3.10 / 3.11 / 3.12) 및 pygame 버전 호환성을 `requirements.txt`로 고정하여 환경 차이로 인한 오류를 예방한다.

- **완화 전략**:
  - Phase 1 MVP 완성 후 2~3일간 직접 플레이 테스트를 통해 조작감과 버그를 점검한다.
  - 각 기능 완성 시마다 Git 커밋으로 되돌릴 수 있는 체크포인트를 유지한다.
